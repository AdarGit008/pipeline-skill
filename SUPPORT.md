# Support

Thanks for using **pipeline-skill**. Here's where to go for help.

## Start with the docs

- **[README](README.md)** — quickstart, exit codes, and how to declare posture.
- **[SKILL.md](SKILL.md)** — the agent surface (score / init / fix / explain).
- **[REFERENCE.md](REFERENCE.md)** — data flow, config keys, CI wiring, and the ledger format.
- **[GLOSSARY.md](GLOSSARY.md)** — plain-language definitions of the terms the rules use.
- **[docs/spec/RULES.md](docs/spec/RULES.md)** — every rule, with a verbatim source quote and URL.

Most "why did this rule fire?" questions are answered by the scorecard's own fix hint
(the `↳` line) and the rule's entry in `RULES.md`.

## Questions & discussion

- **A question, or "is this a bug?"** — open a [GitHub issue](https://github.com/AdarGit008/pipeline-skill/issues)
  using the bug-report or feature-request template. Search existing issues first.
- **A confirmed bug or a new-rule proposal** — file it with the matching issue template so
  it can be tracked and reproduced.

## Security

Do **not** use issues for security reports. Follow **[SECURITY.md](SECURITY.md)** to report a
vulnerability privately via a GitHub security advisory.

## Contributing

Want to fix it yourself? See **[CONTRIBUTING.md](CONTRIBUTING.md)** for the build/test/PR flow.
